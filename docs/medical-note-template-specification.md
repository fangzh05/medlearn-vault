# MedLearn 医学概念笔记模板说明

## 一、模板目标

该模板用于生成可长期维护、可检索、可复习、可追溯的医学概念笔记，保存受来源约束的医学知识，支持学习，并为 Obsidian 提供稳定结构。模板不是要求机械填满全部章节；标题顺序和数据语义必须保持稳定。

## 二、适用对象

适用于疾病、综合征、症状与体征、检查方法、药物、治疗方式、病理过程以及生理或病理机制。疾病类通常使用全部结构；药物、检查和机制类可省略明显不适用章节，不得强行改写成疾病结构。

## 三、信息分层

稳定医学知识、知识关系、学习状态、来源与版本为四个独立层。不得把学习者错误、助手未经验证讲解或模型推断写入稳定医学知识层。

## 四、受控标签规范

标签仅用于稳定分类，不用于关键词、别名、缩写、疾病名称或学习状态。

每篇笔记必须包含：

```yaml
tags:
  - 实体/……
  - 学科/……
```

可选维度仅在明确适用时添加：系统/……、病程/……、临床场景/……、指南/……。固定输出顺序为：实体、学科、系统、病程、临床场景、指南。

实体标签词表：实体/疾病、实体/综合征、实体/症状、实体/体征、实体/检查、实体/药物、实体/治疗、实体/病理过程、实体/机制、实体/解剖结构、实体/病原体、实体/评分工具、实体/指南。

学科标签词表：学科/内科学/呼吸系统、学科/内科学/循环系统、学科/内科学/消化系统、学科/内科学/泌尿系统、学科/内科学/血液系统、学科/内科学/内分泌系统、学科/内科学/风湿免疫、学科/外科学/普通外科、学科/外科学/骨科、学科/外科学/泌尿外科、学科/神经病学、学科/儿科学、学科/妇产科学、学科/药理学、学科/病理学、学科/病理生理学、学科/诊断学、学科/医学影像学、学科/急诊医学、学科/重症医学、学科/肿瘤学。跨学科概念只选择一个主要学科，其余放入 related_disciplines。

系统标签词表：系统/呼吸系统、系统/循环系统、系统/消化系统、系统/泌尿系统、系统/血液系统、系统/神经系统、系统/内分泌系统、系统/免疫系统、系统/运动系统、系统/生殖系统、系统/感觉系统、系统/皮肤、系统/全身性。

病程标签词表：病程/急性、病程/亚急性、病程/慢性、病程/复发性、病程/进展性、病程/自限性；仅在病程是核心属性时使用。

临床场景标签词表：临床场景/急诊、临床场景/门诊、临床场景/住院、临床场景/重症、临床场景/围手术期、临床场景/筛查、临床场景/长期管理。

指南标签词表：指南/GOLD、指南/GINA、指南/ESC、指南/AHA、指南/ACC、指南/KDIGO、指南/NCCN、指南/CSCO、指南/WHO、指南/中国指南。指南版本不得写入标签，应写入 `guidelines`。

禁止：#COPD、#慢阻肺、#重点、#高频、#已掌握、#未掌握、#待复习、#已审核、#最新指南、#难、#常考，以及实体/COPD、实体/慢性阻塞性肺疾病、实体/沙丁胺醇。

## 五、属性字段规范

canonical_name 使用规范中文名；english_name 使用英文全称；aliases 仅保留真实常用同义词、旧称或缩写；concept_type 必须从固定词表选择；external_identifiers 无法确认时为空数组。review_status 仅为 unreviewed、partially_reviewed、reviewed，表示人工复核程度，不表示证据等级。knowledge_status 推荐 active、deprecated、merged、archived；普通有效笔记为 active。

## 六、正文生成原则

先解释因果，再归纳；不得混淆病因与危险因素、症状与体征、筛查与确诊、诊断与严重度、分期与治疗分组、并发症与共病、治疗原则与药物、医学事实与学习者表现。无内容可删除不适用三级标题或写“暂无”，不得编造。数值必须写清人群、场景、单位、指南及重复测量或排除条件；指南敏感内容必须标注版本。

## 七、医学事实来源规则

Composer 输入可包括已有笔记、学习记录、教材或指南片段、黄金样例和固定模板，但权重不同。医学事实仅可来自经确认教材、官方指南或共识、权威数据库、已审核且有来源的现有笔记、已验证结构化 Claim。学习者回答或猜测、即时讲解、无来源模型补充、unverified_chat Claim 和常识推断不得直接作为医学事实。

## 八、学习记录写入规则

“已独立掌握”仅记录学习者未照抄答案而独立正确表达的内容；“对”“懂了”、只选选项或复述完整答案均不算。错误与纠正保留真实错误逻辑；有权威来源支持的纠正标记已验证，否则待验证。待解决问题记录未知、未确认、冲突或待查内容。

## 九、链接规范

优先以规范概念名称链接，例如 [[慢性阻塞性肺疾病]]、[[支气管哮喘]]、[[肺功能检查]]。仅在目标存在或解析唯一稳定时链接；无法确认则保留普通文本并记录待解决问题，不得臆造路径。

## 十、表格使用规范

表格适用于检查比较、鉴别、药物、分期、并发症、风险因素、随访和错误纠正；不适用于连续机制、复杂因果链、大段治疗原则或定义。每列必须有明确的信息维度。

## 十一、空内容处理

无适用内容写“暂无”；未检索到可靠来源写“暂无可靠来源支持”；仍需验证写“待验证”；不适用时删除三级标题。不得用“可能”“一般认为”等掩盖来源不足。

## 十二、Composer 输出约束

只输出完整 Markdown，不输出修改说明、思考过程或免责声明；不得删除仍正确的重要内容，不得将学习错误或未经验证讲解写成结论。保留固定 frontmatter 和稳定章节顺序，标签只用受控词表；不得编造外部编码、指南版本、统计数字、剂量或阈值。全部 Markdown 使用 LF 并以一个换行结尾。

## 十三、推荐的 Composer 输入顺序

1. 系统规则与事实来源规则
2. 当前模板
3. 当前概念的规范元数据
4. 当前已有笔记
5. 本次学习记录
6. 检索出的教材和指南片段
7. 黄金样例
8. 本次生成任务

黄金样例只用于内容密度、解释方式和排版质量，不得复制疾病特异章节或具体医学内容。

## 十四、质量验收标准

合格笔记标题与规范名一致、标签受控、医学正文与学习记录隔离、因果清楚、诊断与评估不混淆、鉴别可操作、数值有条件或来源、指南敏感内容有版本、链接稳定、末尾有来源版本说明，并可直接写入 Obsidian。
