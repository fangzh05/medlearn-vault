# Capture workflow 0.2.0 to 0.3.0

Persistent domain contract 1.2.0 and package version 0.5.1 are unchanged.

Breaking workflow changes:

- remove `CaptureContext.origin`; transport adapters may carry untrusted `client_kind` metadata;
- replace misconception `evidence_message_ids` with required user-owned
  `observed_error_message_ids` and optional `correction_message_ids`;
- derive misconception `observed_at` only from observed-error messages;
- bump `draft_version` and `proposal_version` to `0.3.0`.

There is no implicit compatibility conversion because the old shared evidence list cannot prove
which messages observed the learner error. Producers must classify those references deliberately.
Stored 0.2.0 drafts remain readable as historical bytes but are rejected by current intake and core
validation until explicitly migrated.
