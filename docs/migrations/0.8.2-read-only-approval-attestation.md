# 0.8.2 read-only approval attestation

Package 0.8.2 adds `medlearn workflow verify-approval` and the manually dispatched
`medlearn-verify-approval.yml` workflow. They verify exact existing Approval, Proposal, Job,
Execution, and Review bytes through `medlearn-control` only. The command uses no write method and
does not persist an attestation result.

The manual workflow runs only from `main`: it checks out `main`, exposes its R2 credentials only
to the final read-only verification step, and does not expose them to checkout, setup, or dependency
installation.

This release does not change ProposalApprovalRecord 0.1.0, JobRecord, ProposalExecutionRecord,
CaptureProposal, or persistent medical contracts. It does not authorize publication or replace
future commit-time revalidation. The next sequence is production Approval E2E, followed by PR #15
for deterministic VaultPublicationPlan work; this release has no Vault access, Obsidian action, or
Remotely Save invocation.
