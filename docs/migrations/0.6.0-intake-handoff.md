# Package 0.6.0 intake handoff

Persistent contract 1.2.0 and Capture workflow 0.3.0 are unchanged. IntakeEnvelope is introduced at
0.1.0 and JobRecord advances from 0.1.0 to 0.2.0.

Breaking control-plane changes:

- `draft_digest` becomes `intake_digest` because it hashes exact HTTP envelope bytes;
- `draft_object_key` becomes `intake_object_key`;
- immutable objects move from `v1/drafts/sha256/` to `v1/intakes/sha256/`;
- fixed dispatch inputs use `job_id`, `intake_object_key`, and `intake_digest`;
- JobRecord adds dispatch-attempt and optional lease metadata;
- retries repair missing control objects and retry failed or expired-lease dispatch attempts.

Existing JobRecord 0.1.0 objects are historical records and are not silently rewritten. Existing
idempotency keys should not be reused across the deployment boundary. Deploy the Worker only after
the new package/schema artifacts are available to the future workflow consumer.
