# 0.10.0 — immutable vault writer

## What changed

- Added `VaultPublicationWriter`: reads a verified `VaultPublicationPlan` from
  `medlearn-control`, re-attests its provenance, then writes exact planned
  artifact bytes to `medlearn-vault` R2 with create-only semantics.

- Added `S3VaultObjectStore`: R2 adapter fixed to `medlearn-vault` bucket,
  using independent `VAULT_R2_*` environment variables.

- Added `medlearn workflow publish-vault` CLI command.

- Added `medlearn-publish-vault.yml` GitHub workflow (main-only, scoped
  credentials).

## New environment variables

- `VAULT_R2_ENDPOINT`
- `VAULT_R2_ACCESS_KEY_ID`
- `VAULT_R2_SECRET_ACCESS_KEY`

These must be distinct from the `CONTROL_R2_*` credentials used by existing
workflows. No single step or object holds both credential sets.

## No breaking changes

- Existing `VaultPublicationPlan` 0.1.0 is unchanged.
- Existing control-plane workflows are unchanged.
- No dependency changes.

## What is NOT in this release

- No sync manifest or device synchronisation.
- No Obsidian plugin or integration.
- No Rclone or Remotely Save configuration.
- No D1 database or Durable Objects.
- No VPS or LLM integration.
- No delete, overwrite, or rename semantics.
- No background scheduler or periodic publish job.
