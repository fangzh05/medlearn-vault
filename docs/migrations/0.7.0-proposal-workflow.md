# Package 0.7.0 proposal workflow

Persistent contract 1.2.0, Capture workflow 0.3.0, IntakeEnvelope 0.1.0, and JobRecord 0.2.0 remain
versioned independently. Package 0.7.0 adds orchestration code and ProposalExecutionRecord 0.1.0.

New control keys:

- `v1/executions/<job_id>.json`
- `v1/proposals/<proposal_id>.json`
- `v1/reviews/<proposal_id>.md`

Proposal and review writes are create-only. Existing identical bytes are reused; differing bytes at
the deterministic ID are a collision. Execution leases may be resumed by the same run or taken over
after expiry. This is at-least-once orchestration, and no approval or Vault commit follows it.

JobRecord 0.2.0 semantics are tightened without a version bump: successful terminal records require
proposal/workflow IDs, failures require a stable error code, and terminal states retain no dispatch
lease. Existing invalid records must be repaired explicitly before workflow dispatch.
