# 0.16.1 Windows self-upgrade bootstrap

Windows keeps a running `medlearn.exe` and its Python runtime locked, so a client cannot atomically replace the venv from which it is running.  Version 0.16.1 adds `scripts/install_windows_client.ps1`, a durable external bootstrap that creates a temporary offline installer venv, validates the wheel, and delegates the existing atomic staging/swap/rollback operation to that external client.

Use the bootstrap to upgrade an existing client.  Direct self-hosted upgrades now fail early with `SYNC_INSTALL_SELF_UPGRADE_REQUIRES_BOOTSTRAP`; dry runs and an identical already-installed wheel remain supported.  Configuration, state, and DPAPI credentials stay outside the replaceable venv.
