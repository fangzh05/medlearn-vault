# 0.12.0: Windows Obsidian sync client

This release adds the `medlearn sync configure`, `login`, `logout`, `status`, and `pull`
commands. It is a Windows read-only client for the existing Worker manifest API.

Configuration, cache state, DPAPI ciphertext, and the process lock live outside the Vault at
`%LOCALAPPDATA%\MedLearn\sync\{config.json,state.json,credential.bin,sync.lock}` (overridable
for tests with `MEDLEARN_HOME`). The client verifies canonical manifests, ETags, artifact
digest/length/media type, rejects redirects and manifest rollback, and uses create-only atomic
writes after symlink/junction checks.

It manages only `MedLearn/`, preserves conflicts, and never uploads, deletes, or changes the
Worker, receipts, publication plans, or artifact format.
