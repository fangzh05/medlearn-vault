# Contract migration 1.0.0 to 1.1.0

P0.1 makes stored contracts stricter before any Vault persistence exists.

- IDs are opaque random identities and no longer encode mutable content. Concept, claim,
  relation, and knowledge-unit content now produces a separate computed fingerprint.
- Add semantic scope and `external_identifiers` to concepts; omitted collections deserialize
  to empty tuples.
- Replace claim `truth_value` with `evidence_state` and use the expanded
  `verification_status` vocabulary. Unverified claims default to `unassessed`.
- Update top-level `schema_version` values to `1.1.0`.
- Relations and discipline lenses are independent records; derived reverse references and
  duplicated cross-discipline links are removed from persisted concepts and chapters.
- Misconceptions in captures become immutable observations; lifecycle belongs to a separate,
  rebuildable `LearnerState` projection.
- Source authority moves to `SourceDocument`, and citations use typed locators.
- Naive datetime strings that previously passed validation are now rejected.
- Learning captures and their nested observations are frozen append-only contracts.

No automatic migration is supplied because P0 had no persistent Vault writer or database.
