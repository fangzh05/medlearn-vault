# Contract migration 1.0.0 to 1.1.0

P0.1 makes stored contracts stricter before any Vault persistence exists.

- Existing concept IDs remain unchanged. New IDs must be minted from an immutable identity key;
  names and aliases now produce a separate matching fingerprint.
- Add `external_identifiers` to concepts; omitted fields deserialize to empty lists.
- Replace claim `truth_value` with `evidence_state` and use the expanded
  `verification_status` vocabulary. Unverified claims default to `unassessed`.
- Update top-level `schema_version` values to `1.1.0`.
- Cross-discipline links are now typed objects with four required fields.
- Naive datetime strings that previously passed validation are now rejected.

No automatic migration is supplied because P0 had no persistent Vault writer or database.
