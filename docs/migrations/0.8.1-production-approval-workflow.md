# 0.8.1 production approval workflow

Package 0.8.1 adds `medlearn-approve.yml`, a manually dispatched, control-plane-only runner for
the existing `ProposalApprovalRecord` 0.1.0 contract. It supplies the Proposal ID, exact Proposal
object digest, expected base bundle digest, decision, and optional rejection code to `medlearn
workflow approve`. It adds no contract change: `ProposalApprovalRecord` remains 0.1.0.

The Approval workflow has a 10-minute timeout. Its required `decision` choice defaults to
`approved`; rejected decisions require a `rejection_code`, while the optional input defaults to an
empty string.

The workflow uses only `CONTROL_R2_*` secrets scoped to `medlearn-control`. It has no approval to
write a Vault, invoke Obsidian or Remotely Save, use `medlearn-vault`, or create commits. The new
R2/Remotely Save documents are operational preparation only.
