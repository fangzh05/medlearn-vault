# Contract migration 1.1.1 to 1.2.0

- All contract models are frozen; replace records instead of mutating nested values.
- Remove the ambiguously defined cached `content_hash`; keep `match_fingerprint` for matching.
- Add canonical and scoped ID aliases, `drug_class`, chapter anchors, and capture intervals.
- Move claim conflict handling to `review_status`; superseded claims require replacement IDs.
- Add `ContractBundle`, cross-record integrity validation, `PreviewPlan`, and deterministic Markdown.
