# 0.8.3 synthetic intake workflow

Package 0.8.3 adds a permanent, manually dispatched synthetic intake workflow and the read-only
`medlearn workflow inspect-proposal` command. The workflow accepts no inputs, runs only from
`main`, and submits an excerpt-free fixture through the deployed Worker before reporting exact
Proposal and Review identities.

`MEDLEARN_INGEST_TOKEN` is exposed only to the Worker submission and polling step. The existing
`CONTROL_R2_*` credentials are exposed only to the final read-only inspection step; checkout,
Python setup, and dependency installation receive no credentials. The workflow writes no direct
R2 object, prints no object body, and has no Vault, Obsidian, Remotely Save, D1, VPS, or LLM access.

Persistent and workflow contract versions are unchanged. Approval E2E follows only after this
workflow is reviewed and merged; VaultPublicationPlan work remains postponed.
