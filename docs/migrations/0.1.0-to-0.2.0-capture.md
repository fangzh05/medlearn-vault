# Capture workflow 0.1.0 to 0.2.0

The persistent domain contract remains 1.2.0. Package 0.5.1 changes only the independent capture
workflow contract.

This is an intentional breaking pre-1.0 workflow change. Draft producers must:

- remove `ExtractedClaimCandidate.speaker_role`;
- add `CaptureContext.session_started_at` and `EvidenceMessage.observed_at`;
- add misconception severity;
- add question priority for question assertions;
- emit explicit `learner_evidence_candidates` when a learning outcome was actually observed.

Ownership is derived from evidence-message roles. Missing, mixed, or contradictory assertion
evidence is rejected. `correction_terms` now participate in resolution and correction-claim
matching. Digests preserve ordered arrays and sort only documented set-like fields, so 0.1.0 and
0.2.0 digests are not interchangeable.

There is no silent upgrader because 0.1.0 lacks required event times and explicit learning outcomes.
Existing 0.1.0 drafts must be re-extracted or deliberately supplemented and validated as 0.2.0.
Existing 0.1.0 proposals remain review artifacts but cannot be materialized by the 0.2.0 boundary.
