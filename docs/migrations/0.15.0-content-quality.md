# MedLearn content-quality extension (0.15.0)

This release keeps `LearningCapture.schema_version = 1.2.0` and
`MedLearnHandoff.handoff_version = 0.1.0`. It adds only optional collections, so old captures and
handoffs remain readable and retain their legacy semantic identities when the new fields are empty.

## Optional capture fields

- `assessment_attempts`: immutable question, option, learner-answer, assistant-judgment, feedback,
  concept-link, message-ID, and observation-time context.
- `conversation_explanations`: assistant-stated definitions from learning-chat evidence, always
  marked `unverified_chat`.
- `generated_explanations`: persisted GPT prose with `gpt_generated` origin, `unverified` status,
  generator ID, source learning-session ID, generation time, and context digest.

Generated explanations are separate from claims, learner evidence, and conversation evidence.
Rendering never invokes a model. Rebuilding or Windows syncing reuses the exact persisted text.

## Conversion and rendering

- Handoff conversion moves to `medlearn.handoff_to_intake.v5` and the
  `medlearn-handoff-v5-<digest>` idempotency namespace.
- Renderer `2.1.0` resolves definitions in this order: verified reference, source-backed claim,
  reviewed Chinese catalog scope note, assistant-stated conversation definition, persisted GPT
  explanation, then `暂无解释`.
- The visible concept set is the union of resolved mentions, learner evidence, misconceptions,
  open questions, assessment attempts, and persisted explanations.
- `medlearn capture backfill-capture` creates a new derived Capture from immutable proposal
  explanation statements. It never changes an old Capture or reconstructs missing questions.

Publication plans, Vault writer behavior, R2 create-only semantics, and Windows sync semantics are
unchanged.
