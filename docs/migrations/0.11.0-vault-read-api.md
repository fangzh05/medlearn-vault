# Migration to 0.11.0 — Vault read API

## Summary

PR #19 adds three capabilities:

1. **Immutable `VaultPublicationReceipt`** — deterministic proof of publication stored
   at `v1/publications/<publication_plan_id>.json` in `medlearn-vault`.
2. **Read-only manifest API** — `GET /v1/vault/manifest` lists all published artifacts
   from immutable receipts.
3. **Read-only file download API** — `GET /v1/vault/files?path=...` returns exact artifact
   bytes with digest, byte-length, and media-type integrity verification.

## Version bump

```
0.10.0 → 0.11.0
```

## Schema changes

- New: `VaultPublicationReceipt` 0.1.0 (receipt_version, publication_plan_id,
  publication_plan_object_digest, capture_id, artifacts)
- New: `VaultPublicationReceiptArtifact` (path, media_type, content_digest, byte_length)
- `VaultPublicationResult` gains `receipt_status` field (`created` / `reused` / `none`)
- CLI `publish-vault` output now includes `receipt_status`

## Worker changes

- New `Env` fields: `VAULT_BUCKET` (R2 binding), `MEDLEARN_SYNC_TOKEN` (secret)
- New `wrangler.toml` binding: `VAULT_BUCKET` → `medlearn-vault`
- Vault config is independent of control config; missing vault config only affects vault routes
- New routes:
  - `GET /v1/vault/manifest` — deterministic manifest
  - `GET /v1/vault/files?path=<percent-encoded>` — download with integrity check
- Vault auth uses `MEDLEARN_SYNC_TOKEN` (separate from `MEDLEARN_INGEST_TOKEN`)

## Deployment steps

1. Add `VAULT_BUCKET` R2 binding to `wrangler.toml` (already done in this PR)
2. Set `MEDLEARN_SYNC_TOKEN` Worker secret (≥32 chars, different from ingest token)
3. Deploy Worker with `wrangler deploy`
4. Verify `/health` still returns 200
5. Verify existing intake route still works
6. Create at least one receipt by running `medlearn-publish-vault.yml`
7. Verify `GET /v1/vault/manifest` returns expected artifacts
8. Verify `GET /v1/vault/files?path=...` returns exact bytes with correct headers

## What is NOT in this release

- Windows sync CLI
- Obsidian plugin
- iCloud sync logic
- Local file writing
- Bidirectional sync
- Upload/modify/delete API endpoints
- D1, Durable Objects, KV, Queue, VPS
- WebSocket, Rclone, Remotely Save
- User management, OAuth, Web UI, LLM
- Search index, full-text search
- Git commit auto-write to Vault
- Mutable global manifest object
- Timestamped manifest

## Breaking changes

None. All existing control-plane contracts, workflow inputs, and CLI commands are
backward compatible. `created_count` and `reused_count` semantics are unchanged.
